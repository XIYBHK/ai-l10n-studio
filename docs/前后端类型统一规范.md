# 前后端类型统一规范

## 核心原则

**单一事实来源（Single Source of Truth）**：
- ✅ Rust 端定义类型（后端）
- ✅ ts-rs 自动生成 TypeScript 类型（前端）
- ❌ 禁止前端手动定义重复类型

## 命名规范

### 统一使用 snake_case

```rust
// ✅ 推荐（Rust 标准 + JSON 标准）
pub struct TokenStats {
    pub input_tokens: i32,
    pub output_tokens: i32,
    pub total_tokens: i32,
    pub cost: f64,
}
```

```typescript
// ✅ 自动生成（完全一致）
export interface TokenStats {
  input_tokens: number;
  output_tokens: number;
  total_tokens: number;
  cost: number;
}
```

### 为什么用 snake_case？

| 原因 | 说明 |
|------|------|
| **Rust 标准** | Rust 官方风格指南推荐 |
| **JSON 标准** | 大多数 REST API 使用 snake_case |
| **数据库友好** | SQL 字段名通常是 snake_case |
| **避免转换** | 前后端命名一致，无需映射 |

## 工作流

### 1. Rust 端定义类型

```rust
// src-tauri/src/types.rs
use ts_rs::TS;
use serde::{Serialize, Deserialize};

#[derive(Debug, Clone, Serialize, Deserialize, TS)]
#[ts(export, export_to = "../src/types/generated/")]
pub struct TranslationStats {
    pub total: i32,
    pub tm_hits: i32,
    pub deduplicated: i32,
    pub ai_translated: i32,
    pub tm_learned: i32,
    pub token_stats: TokenStats,
}
```

**关键注解**：
- `#[derive(TS)]`：启用 ts-rs 类型生成
- `#[ts(export)]`：导出为 TypeScript
- `#[ts(export_to = "...")]`：指定导出路径

### 2. 自动生成 TypeScript 类型

```bash
# 运行测试时自动生成类型
cd src-tauri
cargo test

# 生成的文件：src/types/generated/TranslationStats.ts
```

### 3. 前端导入使用

```typescript
// src/types/tauri.ts
export type { TranslationStats } from './generated/TranslationStats';
export type { TokenStats } from './generated/TokenStats';

// 使用时直接导入
import { TranslationStats } from '@/types/tauri';

const stats: TranslationStats = {
  total: 100,
  tm_hits: 30,
  deduplicated: 20,
  ai_translated: 50,
  tm_learned: 10,
  token_stats: { ... },
};
```

## 禁止行为 ❌

### 1. 手动定义重复类型

```typescript
// ❌ 错误示例
export interface TranslationStats {
  total: number;
  tmHits: number;  // 手动定义，与后端不一致
  // ...
}
```

### 2. 手动转换字段名

```typescript
// ❌ 错误示例
const convertedStats = {
  ...stats,
  token_stats: {
    input_tokens: stats.token_stats.prompt_tokens,  // 手动映射
    output_tokens: stats.token_stats.completion_tokens,
  },
};
```

### 3. 使用 any 绕过类型检查

```typescript
// ❌ 错误示例
const stats: any = await translatorCommands.batchTranslate(...);
```

## 前端扩展字段

有时前端需要额外的运行时状态字段，使用接口扩展：

```typescript
// src/types/tauri.ts
import type { POEntry as BasePOEntry } from './generated/POEntry';

export interface POEntry extends BasePOEntry {
  needsReview?: boolean;        // 前端运行时状态
  translationSource?: 'tm' | 'dedup' | 'ai';
  justUpdated?: boolean;        // 动画标记
}
```

**注意**：与后端通信时排除扩展字段：

```typescript
// 保存到后端前，移除前端特有字段
const backendEntry = {
  msgid: entry.msgid,
  msgstr: entry.msgstr,
  msgctxt: entry.msgctxt,
  // 不包括 needsReview、translationSource、justUpdated
};
```

## 常见问题

### Q1: 如何处理 Option<T> 类型？

```rust
// Rust
pub struct MyData {
    #[ts(optional)]
    pub optional_field: Option<String>,
}

// 生成的 TypeScript
export interface MyData {
  optional_field?: string;
}
```

### Q2: 如何重命名字段？

```rust
// Rust 使用 snake_case，TypeScript 也使用 snake_case
// ✅ 推荐：保持一致，不需要重命名

// 如果必须重命名：
#[serde(rename = "customName")]
#[ts(rename = "customName")]
pub custom_name: String,
```

### Q3: 如何处理嵌套类型？

```rust
// Rust
#[derive(TS)]
pub struct Parent {
    pub child: Child,  // Child 也必须 derive(TS)
}

#[derive(TS)]
pub struct Child {
    pub value: i32,
}
```

## 类型一致性检查清单

- [ ] 所有 Rust 公开类型都添加了 `#[derive(TS)]`
- [ ] 运行 `cargo test` 后检查生成的 TS 文件
- [ ] 前端仅从 `src/types/generated/` 导入后端类型
- [ ] 前端扩展字段使用接口继承（extends）
- [ ] 无手动类型转换代码
- [ ] TypeScript 编译无错误（`tsc --noEmit`）
